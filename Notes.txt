Θα γράφω εδώ σκέψεις.

-Δεν ξέρω άμα λειτουργεί η Delete στο array και το sorted array. Λογικά όμως λειτουργεί.
-Είχες αναφέρει το πρόβλημα του τί γίνεται σε περίπτωση που δεν υπάρχει η λέξη που πάμε να διαγράψουμε.

-Ο Παπαδόπουλος στην διάλεξη είχε αναφέρει πως είναι καλό να έχουμε 2 είδη της delete, μία που θα διαγράφει όλη την καταχώρηση και μία που θα μειώνει τον μετρητή των επαναλήψεων της λέξης.

-Έλεγχος αν έχουμε μνήμη, όταν αυξάνουμε τους πίνακες (το κάναμε αυτό χοντρικά).

-Το gutenberg.txt δοκίμασα να το τρέξω στο codeblocks, αλλά δεν τελείωσε ποτέ. Δεν ξέρω αν με κάποιον τρόπο το έκανα pause ή κάτι, αλλά πρέπει να τρέξει...

-Δυστυχώς φαίνεται ότι δεν μπορούμε να χρησιμοποιήσουμε την memcpy για της αντιγραφές πινάκων, η οποία ο Βράκας έχει πει ότι είναι πολύ γρήγορη.

-Στον sorted array ίσως να μπορούμε να έχουμε μόνο μία αναζήτηση, ή την find ή την binarysearch.

-Για το σύνολο Q που θα χρονομετρήσουμε, προτείνω απλά να πάρουμε τις πρώτες 1000 λέξεις του κειμένου, δεν χρειάζεται να κουράσουμε.

-Επισημάνσεις του Παπαδόπουλου "Σωστές και γρήγορες τελικές δομές", "ένα ενιαίο output.txt".

-Στο BinarySearchTree (BST) λειτουργεί η Delete, λογικά και στο AVL.

-Στις δομές των δένδρων δεν μπορεί να υπάρξει μεγάλη κληρονομικότητα όπως θα παρατηρήσεις, γιατί στα ορίσματα δέχονται αντικείμενα των αντίστοιχων κλάσεων, οπότε δεν μπορούν να μπλακούν. Ακόμη και στους κατασκευαστές που έκανα κληρονομικότητα, σκέφτομαι να βγει...

-Όπως θα παρατηρήσεις παίζουν πολλές αναζητήσεις σε κάθε μέθοδο(search, insert, delete), ίσως να μπορεί να γίνει μία για όλες τις μεθόδους; Η search θα μπορούσε να επιστρέφει την διεύθυνση του κόμβου στον οποίο θα πρέπει να προσθέσουμε νέο στοιχείο, αλλά στην main πως θα ξέρουμε το αποτέλεσμα της search, γι' αυτό την έχω bool.
